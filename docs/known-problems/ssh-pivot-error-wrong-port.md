# SSH Pivot Error Shows Wrong Port

**Priority:** P3 (low)
**Found:** 2026-03-25 hardening run (wordpress_server profile)
**Status:** Deferred — needs deeper investigation

## Problem

First SSH attempt from hop 1 to hop 2 showed:

```
ssh: connect to host 172.10.0.11 port 22: Connection timed out
```

after `ssh: failed to open shell: Channel closed`. The error mentions **port 22** but the command used **port 2222**. The attacker noticed the discrepancy.

## Likely Root Cause

The error is generated by Cowrie's SSH transport layer, not the profile. Two possible causes:

1. **Hardcoded default** — the error template may use port 22 instead of reading the actual port from the SSH command arguments.
2. **Race condition** — hop 2 container may not be fully ready when the first connection attempt is made, and the timeout error falls through to a default port value.

## Where to Investigate

- `Cowrie/cowrie-src/src/cowrie/ssh/` — search for "Connection timed out" error templates
- `Cowrie/cowrie-src/src/cowrie/commands/ssh_proxy.py` — check how the target port is passed to the transport layer
- Ensure the actual port from the SSH command (`-p 2222`) is propagated to any error messages

## Workaround

None currently. The issue is cosmetic — the SSH pivot still succeeds on retry.
